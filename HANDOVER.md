# 门店实训志愿分配系统 - 项目交接文档

> 最后更新：2026-07-24

## 一、项目概述

为2026年中国区校招生培训的门店实训环节搭建的志愿分配系统，支持学员在线选择门店志愿，管理员查看分配结果和导出数据。

**在线地址：**
- 学员端：https://store-allocation-production.up.railway.app
- 管理员端：https://store-allocation-production.up.railway.app/admin

**填报时间：** 2026年7月27日 09:00 ~ 2026年7月29日 19:00

---

## 二、业务逻辑

| 项目 | 说明 |
|------|------|
| 学员总数 | 90人（服务类20人 + 非服务类70人） |
| 服务类门店 | 9家（2家各容纳3人，7家各容纳2人） |
| 非服务类门店 | 42家（每家容纳2人） |
| 志愿数量 | 每人最多选3个志愿，按优先级排序 |
| 分配规则 | 按1→2→3志愿顺序分配，第一个有名额的生效 |
| 时间限制 | 未到时间可暂存，正式提交需在规定时间内 |
| 提交后 | 不可修改 |

---

## 三、技术架构

```
前端 React + Vite + TailwindCSS
    ↓
后端 Node.js + Express（运行在 Railway 云平台）
    ↓
数据存储 本地 JSON 文件（Railway 持久化磁盘）
    ↓
飞书同步 通过飞书开放平台 API 增量同步门店和学员数据
```

**GitHub 仓库：** https://github.com/18798055746-lab/store-allocation

## 五、管理员功能

访问 `/admin` 页面，支持以下操作：

| 功能 | 说明 |
|------|------|
| 同步门店 | 从飞书多维表拉取最新门店数据（全量更新） |
| 同步学员 | 从飞书多维表拉取学员数据（增量：新增+更新，不删除） |
| 导入门店 | 上传 Excel（门店名称、可容纳人数、适配范围） |
| 导入学员 | 上传 Excel（工号、姓名、类型、职位、一至五级部门） |
| 清空志愿 | 清空所有学员的分配结果和暂存数据 |
| 清空学员 | 清空所有学员数据 |
| 编辑容量 | 点击门店的"编辑"按钮修改容量 |
| 查看分配详情 | 按服务/非服务分类显示所有门店的分配情况 |
| 学员清单 | 查看所有学员，支持单个添加/删除，显示部门路径 |
| 导出 Excel | 导出分配结果、门店汇总、学员名单三个工作表 |

## 六、学员端功能

## 九、常用操作

### 更新代码并部署

```bash
cd projects/store-allocation
# 修改代码后
npm run build
git add -A
git commit -m "描述"
git push origin main
# 去 Railway 手动点 Deploy
```

### 清空所有分配数据

管理员后台 → 点击"🗑️ 清空志愿"按钮

### 导入真实学员名单

方式一：点击"🔄 同步学员"从飞书自动拉取（增量）
方式二：准备 Excel，点击"📥 导入学员"上传

### 修改填报时间

编辑 `server/index.js` 中的 `ALLOCATION_START` 和 `ALLOCATION_END`

## 十、注意事项

1. **Railway 免费额度**：每月 $5，大概能用 1-2 个月，超了需付费
2. **Railway 部署**：push 代码后需手动在 Railway 点 Deploy 才会更新
3. **数据持久化**：数据存在 Railway 的持久化磁盘，不会因重启丢失
4. **设计风格**：使用不二设计系统（品牌蓝 #2B7FD8、黄 #F4D758、红 #E84A5F）
5. **飞书同步**：学员同步是增量模式（只添加新学员，更新已有信息），门店同步是全量更新
